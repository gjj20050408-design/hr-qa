# 公司HR制度智能问答系统需求调研报告

**文档版本**：V1.0  
**调研日期**：2026年6月  
**调研团队**：HR数字化项目组  

---

## 1. 调研概述

### 1.1 调研背景
随着公司规模扩大与制度体系日益复杂，员工对考勤、薪酬、休假、绩效等政策的咨询量激增。当前主要依赖HR人工解答，存在响应慢、口径不一致、制度查找繁琐、历史问题难以沉淀等问题。为提升员工服务效率、保障制度解释的准确性与及时性，计划建设一套“HR制度智能问答系统”。

### 1.2 调研目的
- 梳理系统目标用户群与核心使用场景
- 明确各角色对系统的功能需求、体验期望
- 为系统架构设计、功能规划和技术选型提供依据

### 1.3 调研对象与方法
- **调研对象**：普通员工（抽样访谈+问卷）、HR共享服务中心专员、HRBP、IT管理员、管理层代表
- **调研方法**：深度访谈、在线问卷、现有工单数据分析、竞品系统参考（飞书智能服务台、钉钉智能HR等）

---

## 2. 现状与痛点分析

| 问题维度 | 具体表现 |
|---------|----------|
| 信息分散 | 制度散落在邮件、共享盘、多个PDF版本中，难以快速定位 |
| 响应延迟 | HR专员需反复查阅文件，平均回复耗时半天以上，高峰时段积压严重 |
| 一致性差 | 不同HR或同一HR在不同时间对同一政策的解释存在差异 |
| 查询门槛高 | 员工难以从长篇制度中快速提取个人相关的权益信息 |
| 知识沉淀弱 | 常见问答记录无序，无法分析员工真实关心的问题分布 |

---

## 3. 系统用户角色定义

| 角色 | 职责与需求概述 |
|------|----------------|
| **普通员工** | 自助查询考勤、休假、薪酬、福利等制度；查看个人专属权益；获取通知 |
| **HR专员** | 上传维护制度文档，管理FAQ，审核纠错反馈，发布通知，查看问答统计 |
| **系统管理员** | 账号权限管理、系统配置、知识库运维、数据安全与审计 |

## 4. 功能性需求明细

依据调研，系统需具备以下11类核心功能，按模块归纳如下：

### 4.1 用户与权限管理（对应需求1）
- **员工账号注册与登录**：支持工号/手机/邮箱注册，集成公司统一认证（SSO/LDAP）
- **部门信息绑定**：账号自动同步组织架构，员工可查看本部门及通用制度
- **权限分级**：
  - 普通员工：仅可查询、提问、查看个人历史
  - HR：制度管理、FAQ维护、通知发布、统计查看
  - 管理员：全系统配置、账号权限分配、日志审计

### 4.2 制度文档全生命周期管理（对应需求2）
- **文档上传与分类**：支持PDF/Word/富文本上传，按“考勤、薪酬、福利、休假、绩效”等标签分类
- **版本控制**：自动保留历史版本，支持版本对比与回滚，更新时提示变更摘要
- **高级检索**：可按标题、分类、发布时间、版本号多维度检索

### 4.3 智能检索与基础问答
- **关键词全文搜索（需求3）**：基于Elasticsearch等实现文档正文精确检索，搜索结果中命中的关键词高亮展示，点击直接定位至原文段落
- **常见问题库（FAQ）维护（需求5）**：HR可对高频问题预设标准答案，分类管理（支持树形分类），员工输入时可快速匹配推荐
- **简单规则问答（需求6）**：基于正则表达式或关键词模板实现基础问答。例：“年假几天” → 匹配制度中“年休假”相关条款，直接返回条文摘要

### 4.4 高级智能问答（对应需求10）
- **检索增强生成（RAG）架构**：将全部制度文档进行向量化处理（Embedding），存入私有向量知识库
- **精准回答生成**：LLM根据用户问题检索相关制度片段，结合上下文生成准确、有出处的答案，并附上参考条文链接
- **多轮追问**：支持上下文对话记忆，可连续追问细节（如“那病假工资怎么算”）
- **跨文档综合推理**：综合考勤制度、薪酬制度、休假制度等多份文档内容，处理复合性问题
- **条件判断问答**：结合员工信息（入职时间、工龄等）进行个性化条件计算。例：“工作满3年的年假天数” → 系统识别工龄≥3年，查询年假梯度规则，返回“您的年假为X天”

### 4.5 制度解读与个性化报告（对应需求11）
- **智能解读**：AI自动将冗长条文生成通俗易懂的图文解读，将流程制度转化为可视化流程图，对比表格展示不同职级/地区的政策差异
- **个性化权益报告**：根据员工个人信息（职级、工龄、地区、家庭状况等）自动生成“您的2026年可享受福利清单”，包含年假天数、体检资格、补贴明细、弹性福利额度等，支持一键导出

### 4.6 个人问答中心（对应需求4）
- **提问历史记录**：完整保存用户所有提问与系统回答，支持按时间、关键词筛选
- **收藏功能**：可收藏常用问答，便于快速复看
- **个人问答日志**：员工可查询自身提问统计（提问频次、常见主题等）

### 4.7 反馈纠错与制度完善（对应需求7）
- **答案准确性反馈**：每次回答下方提供“有帮助/无帮助”按钮，可选填原因
- **制度内容纠错申请**：员工发现制度条文可能过时或错误时，可提交纠错申请并附说明，流转至HR审核后更新文档

### 4.8 通知公告与阅读追踪（对应需求8）
- **HR发布通知**：制度更新、新政策发布时，HR可定向（全员/部门/指定角色）推送公告
- **阅读状态追踪**：记录每位员工是否已读，已读未读列表可视化，支持对未读员工二次提醒

### 4.9 数据统计与分析驾驶舱（对应需求9）
- **问答热度统计**：按日/周/月展示提问总量、有效回答率
- **高频问题分析**：自动聚类提取热点问题标签，识别知识盲区
- **咨询类别分布**：以饼图、柱状图等展示考勤、薪酬、福利等各类问题占比
- **员工画像关联**：可按部门、职级分析咨询偏好，辅助HR主动进行政策宣导

## 5. 非功能性需求

| 指标类别 | 要求 |
|---------|------|
| **性能** | 问答平均响应时间 < 3秒；并发支持 ≥ 500用户同时查询 |
| **可用性** | 系统全年可用性 ≥ 99.9%，支持7×24小时服务 |
| **安全性** | 数据传输加密，敏感个人信息脱敏展示；基于角色的细粒度权限控制；全操作日志审计 |
| **易用性** | 界面简洁，支持移动端自适应；新员工零培训即可完成首次自助查询 |
| **扩展性** | 知识库可接入更多文档类型；可插拔LLM模型层，支持未来升级模型而不影响业务 |

## 6. 可行性分析

### 6.1 技术可行性
- RAG技术栈成熟，LangChain/LlamaIndex配合开源嵌入模型和本地大模型可构建私有知识库，数据不出公司环境
- 全文检索可基于Elasticsearch实现，与权限体系结合稳定
- 流程图与报告生成可调用大模型代码解释能力结合前端可视化库

### 6.2 经济可行性
- 减少HR人工重复咨询量50%以上，预计可释放2-3名专员的工作量用于高价值业务
- 制度维护与发布效率大幅提升，避免因信息滞后造成的合规风险成本

### 6.3 管理可行性
- 公司高层支持HR数字化转型，可协调IT、法务、行政等部门配合
- 需建立制度更新与知识库维护的标准操作流程（SOP），纳入HR日常职责

## 7. 实施建议与风险

**分阶段实施计划建议：**
- **一期**：实现用户权限、制度上传管理、关键词搜索、FAQ及简单规则问答、问答记录
- **二期**：上线基于RAG的智能问答、通知公告与反馈纠错、数据统计
- **三期**：实现制度智能解读、个性化权益报告、高级多条件推理

**主要风险与应对：**
- **答案幻觉风险**：设置回答必须引用原文出处，无匹配时明确提示“未找到相关制度”，并转人工
- **制度更新延迟**：由HR专员负责更新流程，设置版本审核机制，并与通知联动
- **数据隐私**：向量知识库中存储的员工个人画像必须加密隔离，仅用于生成个人报告

---

## 8. 调研结论

本次调研确认，公司各层级对“HR制度智能问答系统”有着强烈且明确的需求，覆盖了从基础检索到智能问答、从被动咨询到主动推送、从标准化回答到个性化解读的全链条场景。所梳理的11项功能需求切中当前业务痛点，技术实现路径清晰，具备较高的投入产出比。建议立项启动，优先聚焦核心问答能力与制度知识库建设，逐步实现智能化升级。
